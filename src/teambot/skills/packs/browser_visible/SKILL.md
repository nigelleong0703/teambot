# 可见浏览器（真实窗口）参考

当前 TeamBot MVP 里的 **browser** 工具已经切到 `browser(action=...)` 协议，但还没有真实可见浏览器 backend。因此当用户明确希望**打开真正的浏览器窗口**、**看到浏览器界面**、**有界面的浏览器**或**可见浏览器**时，应直接说明这项能力尚未在当前 runtime 中实现，而不要伪装成已经支持。

## 何时使用

- 用户说：「打开真实浏览器」「打开有界面的浏览器」「我想看到浏览器」「不要后台，要能看到窗口」
- 用户希望亲眼看到页面加载、点击、填表等过程（演示、调试、教学）
- 用户需要与可见页面交互（如登录、验证码等需人工参与的场景）

## 当前处理方式

1. 直接告知用户：
   - 当前 `browser` 工具只完成了协议拆分
   - 可见浏览器窗口、真实页面交互 backend 仍未接入
2. 如果用户只是给了链接并想读取内容：
   - 改用 `web_fetch`
3. 如果用户明确需要点击、登录、截图、可见窗口：
   - 说明当前 runtime 尚不支持，避免虚构成功结果

## 注意

- 不要再指导用户调用 `browser_use` 或 `headed=true`，因为这不是当前 TeamBot MVP 的真实能力。
- 当真实 browser backend 落地后，再把本 skill 改回具体操作说明。
